# Comparison: 樹Tree vs. 圖Graph

| 比較項目 | 樹 (Tree) | 圖 (Graph) |
|---|---|---|
| 定義 | 一種層級式的結構，由節點與邊組成。 | 由節點（頂點）與連接它們的邊組成的網路。 |
| 環路限制 | 不允許有環路 (Acyclic)。 | 允許有環路，也可以是無環的。 |
| 層級性 | 有明顯的父子關係，且只有一個根節點 (Root)。 | 無根節點概念，頂點間的關係是平等的。 |
| 路徑唯一性 | 任意兩個節點之間只有唯一一條路徑。 | 兩個節點間可能有零條或多條路徑。 |
| 常見類型 | 二元樹 (BST)、紅黑樹、AVL 樹。 | 有向圖/無向圖、加權圖、連通圖。 |
| 常見應用 | 檔案系統、HTML DOM 樹、資料庫索引 (B-Tree)。 | 社交網路、地圖導航 (最短路徑)、網路拓撲。 |

## 什麼時候會用到？

### 使用 Tree 的情境
- 有明確階層關係
- 每個節點只有一個父節點
- 題目出現：hierarchy、root、parent-child

### 使用 Graph 的情境
- 關係複雜、非階層
- 節點可有多條連結
- 題目出現：network、relationship、path

## 總結
- 有沒有 Root？
- 有有沒有 Cycle？
- 節點能不能有多個父節點？

- 都符合 → Tree
- 任一不符合 → Graph
